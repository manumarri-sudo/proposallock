I built a payment-gated file delivery tool after losing $4,200 to a ghosting client

## POST BODY (copy-paste):

18 months ago I handed over final deliverables to a client who had approved every revision, responded to every email, and said payment was "coming end of week."

That was his last email.

I lost $4,200. But the real cost was realizing the entire freelance model is backwards. You do the work, you send the files, then you hope. The moment files transfer, all leverage transfers with them.

I've tracked the time I spend chasing invoices: about 2.5 hours per week. 130 hours a year. Three full work weeks of "hi, just following up on that invoice" emails.

So I built ProposalLock. Payment-gated file delivery for freelancers. Paste a file URL (Google Drive, Dropbox, anything), set a price, send a proposal link. The client sees the project and pays. Files unlock automatically via webhook when payment clears -- not a minute before.

$29 one-time. No subscription. No per-proposal fee.

Live at: https://proposallock.vercel.app?utm_source=indiehackers

Would love honest feedback from the IH community:
- Is $29 one-time the right model vs. $9/month?
- Does "payment-gated proposals" resonate or create friction with clients?
- What features would make you choose this over manually handling payment before sending files?

Stack: Hono + Supabase + LemonSqueezy + Vercel.
